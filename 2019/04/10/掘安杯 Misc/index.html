Misc：

So_easy：

下载后发现在exe下打不开，于是改后缀为txt发现一系列加密字符



于是经过一系列尝试发现是base58转字符串
解码得到



开头为data：image 猜测为图片的base64码
再base64转图片得到一个二维码扫码即可得到flag
flag：jactf{base58_base64_flag_very_easy}

小梳子：

下载后是一个。cap的文件，根据提示猜测需要用字典爆破，
于是在kali中创建字典



然后根据进行爆破



选择1即可跑出手机号


该死的温柔：

查看图片属性提示为：guess

在kali中有outguess的隐写工具：

如果没有：git clone https://github.com/crorvick/outguess

然后在outguess文件夹中进入终端命令：./configure&& make && make install

然后猜测guess为密码对flag.jpg进行解密:
执行命令：outguess -k “guess” -r flag.jpg flag.txt
查看flag.txt即可得到flag


真的不是图片

放在Kali中binwalk发现zip文件但分离不出来



猜测文件被修改过于是在winhex中搜索14000000：因为zip的文件头是504b030414000000



发现504b0304被修改为ja66

修改回来在继续放在kali中binwalk
分离出一个zip的加密文件
密码为ja66打开发现很多文件夹

写脚本：

import re

str1=’’

for i in range(0,32):

str1+=open(‘subject/‘+str(i)+’/‘+str(i)+’.txt’,’r’).read()

print str1

得到：amFjdGZ7NjRzZTY0XzFzXzUwX2MwMDF9

base64解密即可得到flag

你对网站做了什么：

打开shell.pacp文件在里面查看tcp流可以发现是加密操作



又发现了cat/flag.txt加密后的字符


于是写解密操作：
<?php

echo gzuncompress(base64_decode(‘eJxLy0lMrw6NTzPMS4n3TVWsBQAz4wXi’));

?>
即可得到flag

怀疑人生

打开文件由3部分组成

1.第一个是压缩包文件使用字典破解可得到一段base64字符

base64解密后是unicode解码

unicode编码为：\u0062的格式

所以将解出的\u后都加00解码可得第一段flag：flag{hacker

2.是一个图片文件放在kali中binwalk分理出一个zip文件

打开发现需要输密码winhex下发现是zip伪加密

zip伪加密找到： 08 00段将见面改为00 00

打开可得第二段flag：3oD54e

3.是一个二维码文件

用二维码扫描工具扫描可得第三段flag：12580}
